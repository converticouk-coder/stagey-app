## Screenshots — required sizes

Capture in **dark mode** (the app default). Use the device frames Apple/Google
expect. Recommended 6–8 screenshots per platform showing: Home, Shows, Castings,
Society profile, Community, and a tool (Budget Calculator or Rehearsal Pal).

### Apple App Store (required)
| Device class | Resolution (portrait) | Required? |
|---|---|---|
| iPhone 6.7" / 6.9" (15/16 Pro Max) | 1290 × 2796 | **Yes** |
| iPhone 6.5" (11 Pro Max / XS Max) | 1242 × 2688 | Recommended (auto-scales from 6.7") |
| iPad Pro 12.9" (3rd gen+) | 2048 × 2732 | **Yes, only if `supportsTablet` stays true** |

> The app sets `supportsTablet: true`, so a 12.9" iPad screenshot set is required.
> If you don't want to ship iPad screenshots, set `ios.supportsTablet` to `false`
> in `app.json` before submitting.

### Google Play (required)
| Asset | Spec |
|---|---|
| Phone screenshots | 2–8 images, min 1080 px on the short side, 16:9 or 9:16 |
| 7" tablet (optional) | up to 8 images |
| 10" tablet (optional) | up to 8 images |
| Feature graphic | **1024 × 500 PNG/JPG (required)** — purple→pink gradient with the Stagey masks + name |
| App icon | 512 × 512 (Play uses this; generated from `assets/icon.png`) |

> The 512×512 Play icon and the 1024×500 feature graphic are not auto-generated by
> EAS. Export the 512 icon from `assets/icon.png` and design the feature graphic to
> match the brand (see BRAND below).
